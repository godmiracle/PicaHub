## Goal

继续实施 `build-swiftui-picacomic-mvp`，完成 Rasen task 4.1 的 Keychain token store。

## Changes

- 增加领域层 `TokenStore` 协议和可区分的存储错误。
- 增加基于 Security framework 的 `KeychainTokenStore`。
- token 使用 `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`，不允许迁移到其他设备。
- 保存操作优先覆盖已有条目，不存在时创建；删除操作对不存在条目幂等。
- 增加 stub 行为测试与真实系统 Keychain 往返测试。

## Verification

- 完整离线测试集：26 项通过，2 项 credential-gated live test 按设计跳过。
- Keychain 专项测试：9 项在 iPhone Air / iOS 27 上通过，包含真实 Security API 保存、恢复、覆盖和删除。
- `git diff --check`：通过。

## Decisions

- 领域层只依赖 `TokenStore`，不直接依赖 Security framework，后续 AccountRepository 可注入测试替身。
- Keychain 不可用、存储数据损坏和调用方提供空 token 使用不同错误，避免把安全存储故障误报为未登录。
